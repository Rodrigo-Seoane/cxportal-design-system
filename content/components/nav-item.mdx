## Overview

The Nav Item primitives are the building blocks of the sidebar navigation. They are consumed by the `Sidebar` layout component but can also be composed independently when building custom navigation contexts.

There are three exported components:

- **`NavMenuItem`** — 48px group header with leading icon, label, and expand/collapse caret
- **`NavSubItem`** — 40px child route link with left indent, text only
- **`NavMenuItemCollapsed`** — 48×48px icon-only button for the collapsed sidebar state

## When to use

Use these primitives when:
- Building a custom sidebar that differs from the default `Sidebar` layout component
- Rendering nav items in a Storybook story or design reference context
- Extending the navigation with additional item types that follow the same visual language

**Do not use** them to build page-level tabs or breadcrumbs — they are sidebar-specific components.

## Anatomy

### NavMenuItem

- **Leading icon** — 20px, thin weight Phosphor icon. Color inherits from the item's current state.
- **Label** — 14px, Light (300) by default, SemiBold (600) when active.
- **Trailing caret** — `CaretRightIcon` (closed) or `CaretDownIcon` (open). Color matches label.

### NavSubItem

- **Label** — 14px, 48px left indent. No icon. Muted opacity at default, full white when hover/active.

### NavMenuItemCollapsed

- 48×48px square. Icon centered. No label, no caret. Background changes per state.

## Do's & Don'ts

<DosDonts>
  <Do description="Pass state as a prop for static/story use. In the live Sidebar, hover state is derived from React's onMouseEnter/Leave and active from the router pathname — don't drive state from external logic." />
  <Dont description="Use NavMenuItem as a link element. It is a button that toggles a sub-list. Route navigation is handled by NavSubItem links." />
</DosDonts>

<DosDonts>
  <Do description="Use thin-weight Phosphor icons at 20px for NavMenuItem. This is the established visual standard across all CxPortal module icons." />
  <Dont description="Mix icon weights. Filled or bold icons compete with the label text and make the sidebar panel feel heavy." />
</DosDonts>

<DosDonts>
  <Do description="Render NavMenuItemCollapsed only when the sidebar is in its collapsed (64px) state. It replaces NavMenuItem 1:1 and shares the same icon prop." />
  <Dont description="Show NavMenuItemCollapsed alongside visible NavMenuItems. The two forms are mutually exclusive based on the sidebar width state." />
</DosDonts>

## Code

### NavMenuItem

```tsx
import { NavMenuItem } from '@/components/ui/nav-item'
import { BrainIcon } from '@phosphor-icons/react'

// Default
<NavMenuItem
  label="Knowledge Management"
  icon={<BrainIcon size={20} weight="thin" />}
/>

// Active + open (transparent bg, label turns #4285f4)
<NavMenuItem
  label="Knowledge Management"
  icon={<BrainIcon size={20} weight="thin" />}
  state="active"
  isOpen
/>

// Light mode (CxCentral)
<NavMenuItem
  label="Knowledge Management"
  icon={<BrainIcon size={20} weight="thin" />}
  state="active"
  isOpen
  darkMode={false}
/>
```

### NavSubItem

```tsx
import { NavSubItem } from '@/components/ui/nav-item'

<NavSubItem label="Knowledge Bases" />
<NavSubItem label="Add New Document" state="active" />
<NavSubItem label="Folders"          state="disabled" />

// Light mode
<NavSubItem label="Knowledge Bases" darkMode={false} />
```

### NavMenuItemCollapsed

```tsx
import { NavMenuItemCollapsed } from '@/components/ui/nav-item'
import { BrainIcon } from '@phosphor-icons/react'

// Dark (default)
<NavMenuItemCollapsed
  icon={<BrainIcon size={20} weight="thin" />}
  state="active"
/>

// Light
<NavMenuItemCollapsed
  icon={<BrainIcon size={20} weight="thin" />}
  state="active"
  darkMode={false}
/>
```

### Full group (dark)

```tsx
<NavMenuItem
  label="Knowledge Management"
  icon={<BrainIcon size={20} weight="thin" />}
  state="active"
  isOpen
/>
<NavSubItem label="Knowledge Bases"  state="active"  />
<NavSubItem label="Add New Document" state="default" />
<NavSubItem label="Folders"          state="default" />
<NavSubItem label="Tags"             state="default" />
```

## Accessibility

- `NavMenuItem` renders as a `<button>` — keyboard focusable, toggled with Enter or Space.
- `NavSubItem` should be wrapped in a `<Link>` or `<a>` when used for real route navigation (the `Sidebar` component handles this automatically).
- `NavMenuItemCollapsed` renders as a `<button>` and should receive an `aria-label` equal to the group name when used without a visible label.
- The `disabled` state sets the HTML `disabled` attribute, removing the element from the tab order.
- Icon color changes communicate state — ensure the parent container also provides a non-color cue (font weight change) when distinguishing active from default.
