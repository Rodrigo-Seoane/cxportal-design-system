## Overview

Chips and Tags are both compact inline elements, but they serve fundamentally different purposes. **Chips** are interactive — users create, select, filter, or dismiss them. **Tags** are primarily informational — they describe, classify, or indicate the state of content assigned by the system or a user.

## Chip

### When to use

- When users need to make selections from a compact set of options
- For displaying and managing user inputs (such as tags in a form field)
- When filtering or refining content dynamically
- For representing complex entities in a compact form (e.g. a contact with avatar + name + remove action)
- When space is limited and traditional buttons would be too heavy

### Anatomy

- **Container** — 8px border-radius. Padding: 4px vertical, 12px horizontal.
- **Left icon (optional)** — 12px thin-weight icon communicating the chip's domain or type.
- **Label** — 10px SemiBold, 0.4px letter-spacing.
- **Dismiss icon (optional)** — 12px × icon. Clicking removes the chip.

### Variants

#### Type & shade

Chips carry semantic meaning through colour. Each type has four shade levels — 100 (lightest, background-safe) through 500 (darkest, high-contrast).

## Tag

### When to use

- To display metadata about content items
- For showing status or state information alongside content
- When categorising items in lists or tables
- To indicate content type or classification
- For displaying system-assigned or pre-defined labels

### Anatomy

- **Container** — 16px border-radius (pill shape). Padding: 4px vertical, 12px horizontal.
- **Label** — 10px SemiBold, 0.4px letter-spacing.
- **Value (optional)** — Additional numeric or text value shown after the label.
- **Arrow (value-update only)** — 12px thin arrow separating old value from new value.

## Decision framework

**Use a Chip when:**
- The user needs to interact with the element
- The element represents user input that can be modified or removed
- You need selection behaviour (single or multiple)
- The element triggers an action
- You are building a filter interface

**Use a Tag when:**
- The element is primarily informational
- You are displaying metadata or categorisation
- You need to show status or state
- The label is system-defined rather than user-created
- The element describes content rather than controls it

## Do's & Don'ts

<DosDonts>
  <Do description="Use Info shade 100 or 200 for chips embedded in light backgrounds — they provide colour-coded meaning without overwhelming the surrounding content." />
  <Dont description="Use shade 400 or 500 chips in large quantities on a single page. High-contrast shades draw strong attention; reserve them for critical or high-priority states." />
</DosDonts>

<DosDonts>
  <Do description="Always attach an onDismiss handler when the × icon is shown. A dismiss icon with no action confuses users who expect to remove the chip." />
  <Dont description="Show the dismiss icon on chips that are not removable (such as read-only filter labels). Remove iconRight if the chip has no removal behaviour." />
</DosDonts>

<DosDonts>
  <Do description="Use Tags in table cells and list metadata to give users at-a-glance context about each row's state, category, or attribute." />
  <Dont description="Use Tags as primary call-to-action elements. Their low visual weight and read-only intent make them poor substitutes for buttons or chips when an action is required." />
</DosDonts>

<DosDonts>
  <Do description="Use the value-update Tag type to communicate a pending or recently applied change — for example, showing that a count went from 2 to 5 after a sync." />
  <Dont description="Use value-update Tags for static data. This type implies change is in progress or has just occurred; using it for stable values misleads users about the data's state." />
</DosDonts>

<DosDonts>
  <Do description="Keep chip and tag labels short — one to three words. They are scanned, not read, and long labels break the horizontal rhythm of the layout." />
  <Dont description="Truncate chip or tag labels with an ellipsis. If the text doesn't fit, shorten the label or use a tooltip — hidden text defeats the informational purpose." />
</DosDonts>

## Code

### Chip — basic

```tsx
import { Chip } from '@/components/ui/chip'

<Chip label="Current" type="info" shade={100} />
<Chip label="Active"  type="success" shade={200} />
<Chip label="Pending" type="warning" shade={400} />
<Chip label="Blocked" type="error" shade={500} />
```

### Chip — dismissible (controlled)

```tsx
const [tags, setTags] = useState(['Design', 'Research', 'Dev'])

{tags.map((tag) => (
  <Chip
    key={tag}
    label={tag}
    type="info"
    shade={100}
    onDismiss={() => setTags((t) => t.filter((x) => x !== tag))}
  />
))}
```

### Chip — icon variants

```tsx
{/* No icons — label only */}
<Chip label="Draft" type="warning" shade={200} iconLeft={false} iconRight={false} />

{/* Left icon only */}
<Chip label="Connected" type="success" shade={100} iconRight={false} />
```

### Chip — clickable

```tsx
<Chip
  label="Filter by status"
  type="info"
  shade={100}
  onClick={() => openFilterPanel()}
/>
```

### Tag — simple

```tsx
import { Tag } from '@/components/ui/chip'

<Tag label="Published"  state="default" />
<Tag label="Active"     state="active" />
<Tag label="Viewed"     state="viewed" />
<Tag label="Locked"     state="disabled" />
```

### Tag — with value

```tsx
<Tag label="Audience" type="with-value" value="5" state="default" />
<Tag label="Version"  type="with-value" value="v1.2" state="active" />
```

### Tag — value update

```tsx
{/* Shows a change from one value to another */}
<Tag
  label="Count"
  type="value-update"
  value="2"
  newValue="5"
  state="default"
/>
```

## Accessibility

- Chips that are clickable use `role="button"` and `tabIndex={0}` so they receive keyboard focus. `Enter` and `Space` trigger the click handler.
- The dismiss (×) icon uses `role="button"` with `aria-label="Remove {label}"` so screen readers announce its purpose.
- Chips and Tags do not rely on colour alone to convey meaning — always pair a semantic type with a visible text label.
- Disabled Tags carry `aria-disabled="true"` to communicate non-interactivity to assistive technologies.
- When Chips are used as filter toggles in a group, wrap them in a `role="group"` with an `aria-label` that names the filter set (e.g. `aria-label="Filter by status"`).
- For removable input chips (e.g. email recipients), announce removals with an `aria-live="polite"` region so screen readers confirm the item was removed.
