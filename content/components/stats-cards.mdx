## Overview

Stats Cards (also called Data Cards) are compact dashboard components that display key performance indicators at a glance. Each card pairs a large metric value with a category icon, a label, and a trend indicator showing directional change against a reference period — enabling executives to assess business performance in under two seconds without reading detailed text.

## When to use

- Executive dashboards requiring quick KPI assessment
- Campaign performance overviews (delivery rates, response rates, message volumes)
- Resource utilization summaries (credits, capacity)
- Trend comparisons — week-over-week, month-over-month, period-over-period
- High-level status indicators for stakeholders

**Do not use Stats Cards for:**
- Detailed data analysis requiring drill-down — use Tables instead
- Time-series visualization — use Charts instead
- Multi-dimensional comparisons — use Data Tables instead
- Form inputs or data entry — use Form Fields instead

## Anatomy

A Stats Card is composed of four parts:

- **Category icon** — 36×36px (regular) or 24×24px (small) rounded square in primary blue. The icon communicates what metric category this card belongs to (e.g. PaperPlaneTilt for SMS, PhoneCall for Voice).
- **Trend indicator** — A directional arrow (8px) followed by a percentage and period label. Arrow color signals direction: green for increase, red for decrease.
- **Metric label** — The name of the KPI (10px, semibold). Identifies the card when multiple are shown side-by-side.
- **Value** — The primary metric (24px regular / 20px small). The largest element — intended for fast scanning.

## Accessibility

- Use `role="status"` on cards that update dynamically so assistive technology announces changes
- Include a descriptive `aria-label` on the card element — e.g. `"SMS Sent: 6,893, up 5.2% vs last week"`
- Color is never the sole indicator of trend direction — the arrow icon provides a redundant, shape-based signal
- Maintain a minimum **4.5:1** contrast ratio for all text values against the card background
- If cards are interactive (clickable), ensure they receive keyboard focus and display a visible focus ring

## Principles for use

<DosDonts>
  <Do description="Choose the icon that matches the metric category. Category recognition across a row of cards reduces cognitive load when scanning a dense dashboard." />
  <Dont description="Use Stats Cards for metrics that require context or drill-down. If the number needs explanation to be actionable, a table or detail panel is more appropriate." />
</DosDonts>

<DosDonts>
  <Do description="Use the correct trendType — increase for positive, decrease for negative. Consistent color coding trains users to scan by color without reading the text." />
  <Dont description="Omit the trend indicator unless the comparison period is genuinely unknown. A hidden trend removes the most actionable part of the card." />
</DosDonts>

## Code

### Basic usage

```tsx
import { StatCard } from '@/components/ui/stats-cards'

<StatCard
  title="SMS Sent"
  value="6,893"
  trend="5.2% vs last week"
  trendType="increase"
/>
```

### Dashboard row

```tsx
import { StatCard } from '@/components/ui/stats-cards'

<div style={{ display: 'flex', gap: 12, flexWrap: 'wrap' }}>
  <StatCard title="SMS Sent"      value="6,893"  trend="5.2% vs last week"  trendType="increase" icon="sms-sent" />
  <StatCard title="Delivery Rate" value="94.1%"  trend="1.3% vs last week"  trendType="decrease" icon="delivery-rate" />
  <StatCard title="Open Rate"     value="38.7%"  trend="2.0% vs last week"  trendType="increase" icon="open-rate" />
  <StatCard title="Response Rate" value="12.4%"  trend="0.8% vs last week"  trendType="neutral"  icon="response-rate" />
</div>
```

### Small size / blue surface

```tsx
<StatCard
  title="Opt-Out"
  value="142"
  trend="3.1% vs last week"
  trendType="decrease"
  icon="opt-out"
  size="small"
  surface="blue"
/>
```

### Without trend indicator

```tsx
<StatCard
  title="Voice Duration"
  value="4,210 min"
  showTrend={false}
  icon="voice-duration"
/>
```
